《失控視界 Unbounded Vision》

User Story Mapping & Build Order — Team Reference
Engine: Godot 4.6 Core Logic: C (maze_core) Type: 2D 迷宮闖關 / 反獎勵機制 Workshop: user-story-mapping-workshop
核心反轉: 玩家越接近一般遊戲中的成功（拿寶箱、解謎、擴視野），反而越接近失敗。
真正的勝利不是無限擴張，而是學會停止。

Section 0. Workshop Context

Scope & Personas

Scope: 核心 gameplay loop（從進入迷宮到結局；先不管選單 / 設定 / 音效配置）

Personas:

🏆 黑客松評審

需要在 5 分鐘 demo 內看懂技術亮點：C 模組、失控公式可視化、迷宮動態擴張。

🎮 一般玩家

需要自然感受到反轉「aha moment」—— 透過 UI 提示、牆上文字、視野副作用，不是被教學告知。

🔑 關鍵設計決定：離開 loop 的觸發方式

玩家不會一直停在「探索 → 誘惑 → 決定 → 代價」迴圈裡，最終要走向雙出口抉擇。觸發方式選擇了：

✅ 選 b：失控值門檻觸發 當失控值 ≥ 70，C 模組標記「臨界狀態」事件，Godot UI 跳提示「邊界已記住你的貪婪」；假出口開始閃爍但真出口不變。兩個出口都自始至終存在於地圖上，玩家可以隨時走，結局由當下失控值與出口選擇共同判定。

未選的替代方案：a 玩家主動觸發岔路 UI；c 永遠都在（融入探索）。

Section 2. Build Order — Milestones M0 → M6

R0 完成後，R1 的 22 個 task 依下表分入 6 個 milestone。每個 milestone 完成 = 一次 commit + PR + 團隊 review。

M0 — Style Bible（R0） ~ 1 天（硬性 time-box，最多 1.5 天）

目標：產出一份 docs/style-bible.md，讓美術 / 音樂 / UI 隊員「不問你也能做出對的東西」。

  • T0.1Tone Definition（一句話 + 3 參考遊戲）
  • T0.2Voice Guide（整理 PDF 文字、鎖定句式）
  • T0.3Color Palette 5–7 hex + 失控值四階段配色
  • T0.4Visual Style 方向 + Tile 解析度（建議 16×16 或 32×32）
  • T0.5UI Font（中文 + 英文各一，免費可商用）
  • T0.6Music Direction（一句話 + 1 reference 曲）
  • T0.7第一批產出：5-6 個 tile 樣本 + 30s ambient loop
驗收：docs/style-bible.md 所有 TBD 都填完；assets/ 下至少有 floor、wall、player、chest、exit_real、exit_fake 圖檔 + 1 個 ambient OGG
M1 — C ↔ Godot 通訊管線 ~ 1 天

目標：所有後續工作的地基。失敗就 fallback 退守。

  • T2.1C 編譯成 maze_core 執行檔（macOS + Windows）→ Godot 用 OS.execute() 呼叫 → C 輸出 maze_state.json → Godot 讀 JSON 渲染 TileMap
驗收：Godot 啟動 → C 產出 JSON → TileMap 顯示一個隨機迷宮

⚠️ Spike 建議：先花半天驗證 C 編譯 + JSON 通訊在隊員兩種作業系統都能跑。不通就立刻退守 GDExtension 或全 GDScript（會犧牲技術亮點）。

M2 — Player + Vision + First Render ~ 1 天
  • T1.14 向格狀移動 + 撞牆
  • T1.23×3 初始視野迷霧（Shader 或 Light2D）
  • T1.5用 R0 tile 套件渲染（不再是色塊）
  • T2.2走過格子 → 迷霧永久揭開
  • T2.5移動 SFX + 環境 ambient loop（用 R0 產出）
驗收：能在 C 生成的迷宮中走動、有腳步聲、3×3 視野隨玩家移動而揭開
M3 — Objects + Interaction ~ 1.5 天
  • T3.1寶箱 / 鑰匙 / 視野核心 spawn（C 在 JSON 標記座標）
  • T3.2假出口（顯眼大門）+ 真出口（小路）同時 spawn
  • T4.1E 鍵互動偵測
  • T4.2開寶箱 → 視野 +1（3×3 → 5×5 → 7×7 → 9×9）
  • T4.3拿核心 → 視野等級跳級
  • T4.5互動 SFX
驗收：撿 3 個視野核心後，視野從 3×3 變到 9×9；可以看到兩個出口在地圖上
M4 — Instability Core ~ 1.5 天

黑客松靈魂。沒有這個，主題不成立。

  • T1.3HUD 顯示三數值（Vision / Achievement / Instability）
  • T5.1C 模組計算失控值：vision×10 + chests×5 + puzzles×8 + enemies×3 + explored/10
  • T5.2UI 失控值即時更新 + 顏色變化（綠 → 黃 → 橘 → 紅，用 R0 配色）
  • T5.7失控階段警示音（≥31 / ≥61 / ≥81 三段）
驗收：玩家每開一個寶箱，HUD 失控值跳數字 + 顏色變化 + 警示音；按 F1（R2 才實作的 debug overlay）可看公式拆解
M5 — Maze Expansion Feedback ~ 1 天
  • T5.3失控 ≥ 31 → C 模組擴張迷宮（新房間 / 新走道）→ Godot 動態擴 TileMap
驗收：玩家失控值跨過 31 那瞬間，能親眼看到迷宮邊界向外推。失敗條件：擴張卡頓、玩家被擠出 walkable tile
M6 — Endings ~ 1 天
  • T6.1失控 ≥ 70 → C 觸發「臨界狀態」事件
  • T6.2UI 提示「邊界已記住你的貪婪」
  • T7.1結局判定（失控=100 → 壞；走假出口 → 普通；低失控走真出口 → 真）
  • T7.2三種結局文字頁面（文案直接抄 PDF）
  • T7.5三段不同 ending music sting
驗收：玩家三種結局都觸發得到、文字 + 音樂搭配正確。整個遊戲能從頭跑到尾。M6 完成 = 可交件。

Section 3. 並行協作機會

✅ 可並行

  • M0 期間：美術做 tile 草稿 / 音樂家做 ambient loop / PM 寫 voice guide —— 三軌同時跑
  • M1 期間：A 寫 C maze 生成 / B 做 Godot 端 MazeBridge —— 用 JSON 介面切開

  • 同時改 Maze.tscn
  • 同時改 maze_core.c 同一函式
  • 同時改 docs/style-bible.md 同一節（M0 期間用分節分配）

每次平行 session 後跑 /worktree-cleanup（見 CLAUDE.md 規範）。
